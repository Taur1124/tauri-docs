# SvelteKit

本指南将引导您使用 [SvelteKit][] 框架创建您的第一款 Tauri 应用程序。

<Intro />

这是我们将要构建的内容的预览：

![应用预览](/img/guides/getting-started/setup/sveltekit/result.png)

## 创建前端

[SvelteKit][]是一款主要为服务器端渲染（SSR）设计的 Svelte 前端。
为了使 SvelteKit 和 Tauri 正常工作，需要关闭 SSR 并且使用 [`@sveltejs/adapter-static`][] 创建基于前端的静态页面生成 (SSG).

SvelteKit 具有一个与 [`create-tauri-app`][] 类似的脚手架工具，可以根据一些自定义配置快速创建一个新的项目。
对于这个指南，我们会选择 [TypeScript][] 模板，启用 ESLint 和 Prettier。

<Tabs groupId="package-manager">
  <TabItem value="npm">

```shell
npm create svelte@latest
```

  </TabItem>

  <TabItem value="Yarn">

```shell
yarn create svelte
```

  </TabItem>

  <TabItem value="pnpm">

```shell
pnpm create svelte
```

  </TabItem>
</Tabs>

1. _项目名称_ <br />
   这将是您的 JavaScript 项目的名称，对应此工具将创建的文件夹的名称，但在其他方面对您的应用没有影响。您可以在此处填写任何您想要的名称。

2. _应用模板_ <br />
   我们会选择 `Skeleton project` 以创建一个裸露的模板。如果您想要使用一个更完整的 SwelteKit 示例体验，您可以选择 `SwelteKit 演示应用程序`。

3. _输入检查_ <br />
   您是否想要在您的项目中通过 JSDoc 或 TypeScript 进行类型检查。 对于本指南，我们假设您选择 TypeScript。

4. _代码检查和格式化_ <br />
   您是否是想在项目启动时使用 ESLint 进行代码检查和使用 Prettier 进行代码格式化。在本指南中不会有关于这个问题的其他提及，但我们建议启用这两个选项。

5. _浏览器测试_ <br />
   SwelteKit 为浏览器测试提供内置的 Playwright 支持。既然 Tauri API 不适用于 Playwright，我们建议不要添加它。查阅我们的 [WebDriver 文档][WebDriver documentation] 以获取对于 Selenium 或 WebdriverIO 而非 Playwright 的替代方案。

## SvelteKit - SSG 模式

首先，我们需要下载 [`@sveltejs/adapter-static`][]：

<Tabs groupId="package-manager">
  <TabItem value="npm">

```shell
npm install --save-dev @sveltejs/adapter-static@next
```

  </TabItem>

  <TabItem value="Yarn">

```shell
yarn add -D @sveltejs/adapter-static@next
```

  </TabItem>

  <TabItem value="pnpm">

```shell
pnpm add -D @sveltejs/adapter-static@next
```

  </TabItem>
</Tabs>

然后更新 `adapter` 并导入到 `svelte.config.js` 文件：

```javascript title=svelte.config.js
// highlight-next-line
import adapter from '@sveltejs/adapter-static' // 这将会通过 adapter-auto 更改
import preprocess from 'svelte-preprocess'

/** @type {import('@sveltejs/kit').Config} */
const config = {
  // 查阅 https://github.com/sveltejs/svelte-preprocess 以了解更多关于预处理器的信息
  preprocess: preprocess(),

  kit: {
    adapter: adapter(),
  },
}

export default config
```

最后，我们需要禁用 SSR 并通过添加包含以下内容的根 `+layout.ts` 文件（如果您不使用 TypeScript，则为 `+layout.js`）来启用预渲染：

```typescript title=src/routes/+layout.ts
export const prerender = true
export const ssr = false
```

请注意，static-adapter 不要求您对整个应用程序禁用 SSR，但它可以使用依赖于全局 `window` 对象的 API（如 Tauri 的 API），而无需[客户端检查][Client-side checks]。

此外，如果您喜欢单页应用程序模式（SPA）而不是 SSG，您可以根据 [适配器文档][] 更改适配器配置和 `+layout.ts`。

现在我们已经搭建好了前端并初始化了 Rust 项目，创建的 `tauri.conf.json` 文件应该如下所示：

```json title=src-tauri/tauri.conf.json
{
  "build": {
    // 当您运行 `tauri build` 时，将会执行这个命令。
    "beforeBuildCommand": "npm run build",
    // 当您运行 `tauri dev` 时，将会执行这个命令。
    "beforeDevCommand": "npm run dev",
    "devPath": "http://localhost:5173",
    "distDir": "../build"
  },
  ...
}
```

## 调用指令

<Commands />

使用 [`@tauri-apps/api`][] JavaScript 库来调用新创建的命令， 通过 JavaScript 访问诸如窗口、文件系统等核心功能， 您可以使用自己喜欢的 JavaScript 包管理器来安装。

<InstallTauriApi />

安装依赖库后，我们现在可以创建一个新的 Svelte 组件。 我们将在 `src/lib/Greet.svelte` 中创建它：

```html title=src/lib/Greet.svelte
<script>
  import { invoke } from '@tauri-apps/api/tauri'

  let name = ''
  let greetMsg = ''

  async function greet() {
    greetMsg = await invoke('greet', { name })
  }
</script>

<div>
  <input id="greet-input" placeholder="Enter a name..." bind:value="{name}" />
  <button on:click="{greet}">Greet</button>
  <p>{greetMsg}</p>
</div>
```

您现在可以将此组件添加到 `src/routes/+page.svelte`:

```html title=src/routes/+page.svelte
<script>
  import Greet from '../lib/Greet.svelte'
</script>

<h1>Welcome to SvelteKit</h1>
<Greet />
```

现在您可以运行 `npm run tauri dev` 并查看结果：
